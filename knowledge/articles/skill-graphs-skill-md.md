## Summary

Skill Graphs > SKILL.md proposes a new paradigm for organizing complex knowledge and capabilities through interconnected, composable skill files rather than monolithic skill documents. Instead of capturing one aspect of something in a single file, this approach uses skill graphs—networks of skill files connected with wikilinks—to enable entirely new kinds of applications and deeper knowledge representation.

The article argues that real depth in skill systems requires breaking down large capabilites into many small, composable pieces that reference each other through meaningful wikilinks woven into prose. This creates a traversable graph where every node has a YAML description that agents can scan without reading entire files, enabling progressive disclosure from index descriptions to full content.

## Key Takeaways

- **Structured Knowledge Power**: Structured knowledge enables entirely new kinds of applications beyond what flat files can support
- **Network Over Files**: A skill graph is more powerful than a single skill file because it captures relationships and dependencies
- **Progressive Disclosure**: Index → descriptions → links → sections → full content hierarchy allows agents to make decisions before reading full files
- **YAML Frontmatter**: Metadata in each file enables agent-readable scanning without full content access
- **Meaningful Links**: Wikilinks that carry semantic meaning through prose are more useful than bare references
- **Recursive Application**: Skill discovery patterns can be applied recursively within graphs themselves
- **Already Have Primitives**: Wikilinks, YAML frontmatter, and markdown files provide everything needed to implement skill graphs
