## Summary
A detailed guide on implementing a Subconscious agent pattern that enables autonomous agents (Hermes, OpenClaw) to continuously self-improve and compound their capabilities. Instead of producing more output, the Subconscious agent identifies patterns, debates improvements, and writes changes back into the system so each subsequent run starts smarter.

## Key Takeaways
- **Subconscious vs Output**: Most agent systems fail by confusing output with improvement. The Subconscious agent focuses on identifying what actually compounds vs what becomes stale.
- **Pattern Recognition**: The agent continuously asks: What keeps returning across multiple runs? What's a one-off impulse? What was built and still mentioned later? What deserves experimentation?
- **Architecture**: Inside Hermes, the Subconscious has its own isolated profile with dedicated workspace, memory, schedule, habits, project shelf, feedback loop, and hard boundaries.
- **Self-Improvement Loop**: This enables agents to brainstorm, debate, refine, and write improvements back into the system for the next run.
- **Isolation is Key**: Isolation prevents context pollution and enables coherent long-term specialization of agent profiles.
