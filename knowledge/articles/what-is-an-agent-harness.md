A foundational clarification on what an "agent harness" actually means, separating it from agent frameworks like LangChain and LangGraph.

## Key Takeaways

- **Harnesses vs. Frameworks**: LangChain and LangGraph are human-oriented frameworks with dozens of configuration knobs; harnesses are different systems designed for agents to operate autonomously
- **Design Philosophy Difference**: Frameworks assume a human architect configures components correctly; harnesses are built bottom-up from practical agent work, not top-down from abstractions
- **The Industry Confusion**: The term "harness" is widely used but poorly understood; the post clarifies what distinguishes a true harness from earlier-generation agent frameworks
- **Missing Definition**: Common quotes like "if you're not the model, you're the harness" are vague and obscure rather than clarify the core difference
