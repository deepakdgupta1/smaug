## Summary
Instead of asking AI for 100x ideas, build a system to hunt for real pain in public conversations. The gold is in social cues that reveal expensive, repeated problems people are already complaining about and paying to solve badly.

## The Framework
Build this system:
```
Social listening alerts
→ database
→ AI pain extraction
→ opportunity scoring
→ manual MVP
→ build/pivot/kill
```

## Pain Signals to Listen For
The data reveals itself through specific complaint patterns:
- "I hate doing this manually."
- "Does anyone know a tool for this?"
- "Is there an alternative to this?"
- "This is too expensive."
- "I'm still using a spreadsheet."
- "I waste hours on this every week."
- "Why is this so hard?"
- "I tried X and Y and Z, but it still sucks."

## Best Sources: Reddit vs X

**Reddit:**
- Long, emotional complaints
- Detailed problem explanations
- Workaround documentation
- Failed tool attempts
- Edge case articulation
- Emotional investment clarity

**X:**
- Public complaints and requests
- Recommendation-seeking
- Tool comparisons
- Real-time trend signals
- Community validation

## Key Insight
Don't build what AI hallucinates. Hunt where people already complain. The signal is strongest where pain is repeated, expensive, and actively being worked around.

## Implementation
1. Set up social listening on Reddit and X
2. Extract pain signals using AI
3. Score opportunities by repeatability and cost
4. Build minimal MVP
5. Test with real users
6. Iterate based on validation
