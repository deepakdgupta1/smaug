## Summary

A practical guide to a `/research` skill that prepares context before starting any coding task in Claude Code. While Claude Code's Plan mode is effective, it has knowledge cutoffs and gaps. The skill ensures tasks have the latest documentation and web search results, solving problems more efficiently.

The skill leverages two key integrations: Context7 MCP for accessing documentation (especially useful for JavaScript-heavy or authentication-protected sites), and ui.sh toolkit for design tasks (optional). The research process includes clarifying the topic, searching docs, web, and codebase in parallel, then synthesizing findings into actionable context.

## Key Takeaways

- **Pre-planning Research**: Run deep research before planning, not during execution
- **Parallel Search**: Launch agents to search docs, web, and codebase simultaneously
- **Documentation Gap**: Many developer sites are JavaScript-heavy or behind auth—Context7 fills this gap efficiently
- **Web Search**: Recent information and implementations that won't be in training data
- **Synthesis**: Combine findings into actionable context before planning starts
- **Harness Agnostic**: Adaptable to other harnesses beyond Claude Code
